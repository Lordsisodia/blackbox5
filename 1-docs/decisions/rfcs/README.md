# Requests for Comments

> RFCs for proposed changes and features

## Overview

This directory contains Requests for Comments (RFCs) documenting proposed changes, new features, and significant enhancements to BlackBox5.

## RFC Lifecycle

1. **Draft** - Initial proposal
2. **Proposed** - Ready for review
3. **Accepted** - Approved for implementation
4. **Implemented** - Code complete
5. **Rejected** - Not moving forward

## Current RFCs

| RFC | Title | Status |

| RFC-001 | Modular Agent System | Implemented |
| RFC-002 | Skill Registry | Implemented |
| RFC-003 | Context Optimization | Accepted |
| RFC-004 | Distributed Execution | Proposed |

## RFC Template

RFCs should include:
- Summary
- Motivation
- Detailed design
- Drawbacks
- Alternatives
- Unresolved questions

## Creating RFCs

1. Use `rfc-template.md`
2. Fill in all sections thoroughly
3. Submit PR for discussion
4. Address feedback
5. Await decision
