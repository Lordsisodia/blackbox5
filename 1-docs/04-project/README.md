# 04 - Project

This section contains project management, planning, and coordination documentation.

## Subsections

### 01-roadmap/
Project roadmaps, milestones, and future plans.

### 02-milestones/
Milestone definitions, deliverables, and tracking.

### 03-sprints/
Sprint planning, retrospectives, and iteration docs.

### 04-backlog/
Feature backlog, issue tracking, and prioritization.

### 05-releases/
Release notes, version history, and changelog.

## Purpose

Project documentation coordinates team efforts:
- Where we're going (roadmap)
- What we're doing now (sprints)
- What's coming next (backlog)
- What we've accomplished (releases)

## Key Documents

- Roadmap and vision documents
- Sprint plans and retrospectives
- Feature specifications
- Release planning

## Contributing

When adding project docs:
1. Keep status information current
2. Use dates and versions
3. Link to related decisions in `decisions/`
4. Update when plans change
